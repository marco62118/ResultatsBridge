Présentation générale

ResultatsBridge Un véritable boitier de marque sur les tables de tournoi de bridge dans un seul smartphone par table
Pas de matériel coûteux, pas d'installation complexe.

L'application fonctionne sur tout smartphone Android (version 6 minimum) et couvre l'ensemble du cycle de vie d'un tournoi : constitution des équipes, choix du type de tournoi, suivi des mouvements, enregistrement des donnes, calcul automatique du classement et consultation des résultats en local ou en ligne.

🏆 Classement automatique : calculé et affiché immédiatement et automatiquement en fin de tournoi.
🃏 Saisie pédagogique : mains saisies une seule fois et enchères enregidtrées donne par donne pour analyse après tournoi.
📡 Local ou en ligne : mode Wi-Fi interne sans Internet, ou mode en ligne.
📤 Export en un clic : l'organisateur publie le tournoi sur le serveur en ligne directement depuis l'application.
🏛 Espace club : classements, résultats détaillés de chaque donne, de chaque équipe consultables pour tous les tournois exportés.

Modes de fonctionnement

📡 Mode local Wi-Fi : L'Organisateur crée un point d'accès Wi-Fi sur son smartphone ou utilise un réseau Wi-Fi présent. Les joueurs s'y connectent et saisissent l'adresse IP communiquée par l'Organisateur. Aucun Internet requis. Export vers le serveur possible en fin de tournoi.
🌐 Mode en ligne : L'Organisateur crée le tournoi directement sur le serveur. Les saisies sont envoyées en temps réel vers le serveur. Résultats accessibles immédiatement par les adhérents.

Préparation d'un tournoi

L'Organisateur configure le tournoi depuis son smartphone : nombre d'équipes, de tables, de donnes par table et de type le tournoi.

Types de tournois supportés

  • Tournois par 4 équipes sur 2 tables
  • Tournois « Howell » avec ou sans table relais
  • Tournois « Mitchell »

Tous les mouvements sont préétablis et toutes informations nécessaires sont affichées sur chaque smartphone de table, assurant le bon déroulement.

Les équipes sont constituées rapidement depuis la liste des adhérents enregistrés. L'ajout de nouveaux joueurs est possible avant le lancement. L'Organisateur peut lui-même participer au tournoi depuis son propre smartphone.

Table relais : lorsqu'une table relais existe (tournoi Howell), l'application le signale et invite les joueurs à saisir les mains des donnes non jouées — un gain de temps précieux pour les autres tables.

Déroulement du Tournoi

À chaque table, un seul des joueurs — le marqueur — manipule le smartphone.
Il ouvre l'application, sélectionne son équipe, et vérifie : équipe adverse, numéro de table, donnes à jouer.
La partie peut commencer.

Saisie des enchères

A la fin des enchères, le marqueur, en commençant par Nord, enregistre toutes ses enchères (pass, 1C, pass) puis Est (pass, pass, pass) puis Sud (1T, 1SA, pass) Ouest( pass, pass, pass). Le contrat final s'affiche automatiquement.

💡 si vous ne souhaitez pas saisir le détail des enchères, vous pouvez entrer directement le contrat final ⚠️ vous perdrez cependant la comparaison inter-tables.

Carte d'entame

L'entameur pose sa première carte. Le marqueur la saisit immédiatement
⚠️ étape obligatoire pour pouvoir enregistrer les résultats et continuer le tournoi.

Saisie des résultats

La donne terminée, clic sur . Un résultat par défaut (= 0) s'affiche avec les points. Le marqueur modifie le résultat si nécessaire
Les points sont recalculés automatiquement. ⚠️ Un résultat incohérent (= 1 ou + 0) bloque l'affichage des points.

Vérification des mains

Clic sur pour s'assurer que tout le monde a joué les mêmes cartes (contrôle des étuis).

  • Si une autre table a déjà saisi les mains, une vérification sommaire suffit.
  • Sinon : saisie des cartes de Nord → Est → Sud. Les 13 cartes restantes vont à Ouest automatiquement.
  • ⚠️ Sans saisie des mains : pas de comparaison inter-tables possible après le tournoi.

Donne suivante

Le clic sur enregistre tout définitivement sur le serveur. Continuer ainsi jusqu'à la dernière donne de la table.

Changement de mouvement

À la dernière donne, l'écran « Changement de mouvement » indique la prochaine table et l'adversaire à rencontrer.

⏳ Attendre le feu vert de l'Organisateur avant de changer de table.

Fin de tournoi et classement

Au dernier mouvement, dès que toutes les équipes ont enregistré leurs donnes, le classement est calculé automatiquement et affiché sur tous les smartphones connectés.

Le clic sur

Un tableau détaillé liste toutes les donnes jouées par chaque équipe. En cliquant sur une ligne, on visualise les 4 mains et les enchères des deux équipes, si les saisies ont été effectuées.

L'Organisateur peut ensuite exporter le tournoi vers le serveur en ligne en un seul clic.

1. Points Match (PM)

Les Points de Match (PM) servent à classer les résultats de manière relative : on ne compte pas l'écart de points au score, mais le nombre d'équipes que l'on a battues sur une même donne.

Grille de calcul des PM

Comparaison du score Points attribués (PM)
Votre score est supérieur à celui d'une autre équipe2 PM
Votre score est égal à celui d'une autre équipe (partage)1 PM
Votre score est inférieur à celui d'une autre équipe0 PM

Le Top par donne

Le Top est le score maximum qu'une équipe peut obtenir sur une donne jouée n fois.

Top = 2 × (n − 1)
  • Donne jouée 2 fois : Top = 2 PM
  • Donne jouée 3 fois : Top = 4 PM

2. Points Neuberg (Ajustement)

La formule de Neuberg corrige les déséquilibres lorsqu'une donne n'a pas été jouée le même nombre de fois par toutes les équipes (en raison d'un relais ou d'un incident).

PM ajusté = [ ( (PM obtenu + 1) × N ) / (n + 1) ] − 1

N : Nombre de fois maximum où la donne a été jouée dans le tournoi.
n : Nombre de fois effectif où cette donne spécifique a été jouée.

Exemple d'ajustement (Base N = 6)

n (fois jouées) PM réel obtenu PM après Neuberg
6 (complet)88,00
4711,00
325,00
301,00

3. Mitchell avec Skip (Nombre de tables pair)

Le problème de la symétrie

Dans un Mitchell standard à nombre de tables pair (ex: 4, 6, 8), les équipes Est-Ouest finiraient par rejouer les mêmes donnes à mi-parcours.

La solution du saut (Skip)

Après la moitié des mouvements, les équipes Est-Ouest effectuent un saut en ignorant la table suivante (+2 tables au lieu de +1). Cela casse la symétrie et évite les doublons.

Avantage Inconvénient
Mouvement simple et rapide. Les équipes EO jouent un set de donnes en moins (une rencontre de moins).

3. Tournoi Mitchell avec Guéridon

Le rôle du Guéridon

Le guéridon désigne l'emplacement où un set de donnes n'est pas joué durant une ronde. C'est associé avec un partage de set de donnes entre 2 tables. Ce système permet de gérer un nombre de tables pair sans avoir recours au "Skip" (saut de table) et donc de jouer tous le même nombre de donnes.

Fonctionnement du partage et du repos

Élément Règle de mouvement
Le Partage La table 1 et la dernière table jouent simultanément le même set de donnes et se les échangent pendant la ronde.
Le Guéridon Contient le set de donnes qui n'est pas joué par personne pendant la ronde actuelle. Il sert de zone de transit pour la rotation suivante.
Équipes EO Se déplacent normalement d'une table vers le haut à chaque ronde.
Les set de donnes Se déplacent normalement d'une table vers le bas à chaque ronde, table Guéridon compris.
Équipes NS Restent fixes à leur table.

Avantages du système

  • Permet de jouer toutes les rondes (pas de perte de rencontre comme avec le Skip).
  • Garantit que les équipes EO ne rencontrent jamais deux fois les mêmes donnes.
  • Idéal pour intégrer une équipe "Relais" si le nombre total d'équipes est impair. La dernière table sera la table Relais et la table 1 ne sera plus obligée de partager son set de donnes, la table relais ne jouant pas de donnes

4. Table Relais

💡 Équipe Relais : Si le nombre total d'équipes est impair (ex: 7 équipes pour 4 tables), l'équipe EO arrivant à la place "Relais" ne joue pas pour la ronde. Dans ce cas en Mitchell les EO jouent un set de moins que les NS
