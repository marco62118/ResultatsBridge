1. Points Neuberg

Pourquoi le Neuberg ?

Dans un tournoi de bridge, chaque donne est normalement jouée le même nombre de fois par toutes les équipes. Mais parfois, une ou plusieurs donnes n'ont pas été jouées par certaines équipes (table Relais, temps, retard, donne sautée…). Elles ont alors joué moins de donne que les autres équipes.

Imaginez deux donnes :

  • Donne A : Jouée 3 fois. Top possible = 4 pts en PM.
  • Donne B : Jouée 2 fois seulement. Top possible = 2 pts en PM.

Si une paire NS marque le Top sur la Donne B, elle ne gagne que 2 points, alors que celle qui gagne la paire NS Donne A en marque 4. Il y aura bien une différence sur le classement total. C'est injuste car la performance est la même (être premier).

La formule de Neuberg corrige ce déséquilibre en "gonflant" les PM des donnes moins jouées pour les ramener à la moyenne du tournoi.

La formule de Neuberg

PM ajusté = ( N × (PM + 1) ) / n − 1
  • N = nombre total de fois que la donne a été jouée (nombre maximum)
  • n = nombre de fois que cette équipe a joué cette donne
  • PM = points obtenu en Points Match avant ajustement

Exemple concret

Supposons une donne jouée N = 6 fois au total. Une équipe NS l'a jouée n = 4 fois et a obtenu 7 points en PM simplifiés (PMtop = 10).

PM ajusté = ( 6 × (7 + 1) ) / 4 − 1 = 48 / 4 − 1 = 11

💡 La correction Neuberg n'est appliquée que lorsque n < N, c'est-à-dire quand une donne n'a pas été jouée le nombre maximum de fois. Si n = N, le score reste inchangé.

Score en pourcentage (%)

En plus du score en points, l'application affiche le score en pourcentage pour faciliter la comparaison :

Score % = ( points obtenus / (donnes jouées × Ntop) ) × 100

Où Ntop = (N − 1) × 2 est le score maximum possible sur une donne (obtenu si l'on bat tout le monde).

Exemple : 45 points sur 60 possibles → 75,0 %

2. Tournoi Mitchell avec skip (nombre de tables pair)

Rappel : le mouvement Mitchell standard

Dans un tournoi Mitchell, les paires NS restent à leur table d'un mouvement à l'autre. Les paires EO se déplacent d'une table vers le haut (table 1 → 2 → 3…) à chaque mouvement, et les donnes se déplacent d'une table vers le bas.

Le problème avec un nombre pair de tables

Quand le nombre de tables est pair (ex. : 4, 6, 8 tables), le mouvement Mitchell classique crée une situation où chaque paire EO rencontre exactement les mêmes paires NS qu'une autre paire EO. Certaines rencontres sont donc dupliquées — deux paires EO jouent exactement le même chemin — ce qui rend le classement inéquitable.

La solution : le skip

Après exactement la moitié des mouvements, les paires EO font un saut de deux tables au lieu d'une. Ce "skip" casse la symétrie et garantit que toutes les paires EO affrontent des paires NS différentes.

⚠️ Le skip se produit automatiquement. L'application affiche un avertissement en rouge "⚠️ sauté à la Table X" pour signaler ce mouvement spécial.

Exemple : 6 tables, 11 mouvements

Le skip a lieu après le mouvement 3 (= 6 ÷ 2). La paire EO saute 2 tables au lieu d'une.

💡 Pour un nombre impair de tables (3, 5, 7…), aucun skip n'est nécessaire : le mouvement Mitchell classique garantit déjà que toutes les rencontres sont uniques.

3. Tournoi Mitchell avec guéridon

À quoi sert le guéridon ?

Le guéridon (aussi appelé "table fantôme" ou "relay table") est une table virtuelle utilisée lorsque le nombre de paires est impair. Sans lui, une table resterait incomplète à chaque mouvement (une paire sans adversaire).

Le guéridon permet d'accueillir les donnes en attente pendant qu'une paire est en "bye" (au repos ce mouvement-ci). Chaque paire passe par le guéridon exactement une fois.

Comment fonctionne le guéridon ?

Prenons l'exemple d'un tournoi à 5 tables (10 paires — nombre pair) mais avec 1 paire de moins = 9 paires. On crée alors 5 tables dont une sera le guéridon.

🪑

Le guéridon : table d'attente des donnes

Règles de mouvement avec guéridon

  • Les paires NS restent à leur table, comme dans un Mitchell classique.
  • Les paires EO se déplacent d'une table à la suivante à chaque mouvement.
  • Les donnes se déplacent aussi, mais dans le sens opposé aux EO.
  • Quand une paire EO arrive au guéridon, elle ne joue pas ce mouvement — elle attend que les autres terminent.
  • Les donnes présentes au guéridon à ce moment-là "sautent" également.

Avantages

  • Permet de jouer avec un nombre impair de paires sans exclure personne définitivement.
  • Pas de skip dans ce format : le mouvement est régulier (+1 table à chaque fois).
  • Toutes les paires jouent le même nombre de donnes au final.
💡 Dans l'application, la paire "Relais" (Guéridon) est créée automatiquement. Elle n'a pas de vrais joueurs — son nom est RELAIS. Quand l'équipe Relais est adversaire, la paire EO attend à la table relais jusqu'au mouvement suivant.
